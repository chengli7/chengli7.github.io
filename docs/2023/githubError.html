github无法访问解决方式

# 问题描述

访问GitHubopen in new window时，总是无法访问，例如出现如下情况： 如果无法访问GitHub，很有可能是由于网络层面的问题。可能存在以下问题： DNS是一种用于将网址转换为IP地址的工具，如果电脑无法访问GitHub，那么很有可能是因为DNS出现了问题

# 解决方案

# 方式一. 修改hosts地址

  1. 获取域名IP 可以进入这个网址： IP/服务器github.com的信息 - 站长工具获取IP地址open in new window获取github域名的ip地址
  2. 修改hosts文件

通过上述方式获取IP地址之后

（1）如果电脑是Windows系统：打开 C:\Windows\System32\drivers\etc 找到hosts文件，将对应的Host地址修改为：

#github
140.82.112.4 github.com

更改完成后，打开命令提示符（cmd），刷新DNS

ipconfig /flushdns

（2）如果电脑是Linux系统：vim /etc/hosts打开hosts文件，添加如下内容：

#github
140.82.112.4 github.com

更改完成后，刷新DNS

sudo systemctl reload NetworkManager
或者
sudo systemctl restart NetworkManager

若上述方式未解决，可尝试用方式二解决

# 方式二. 安装dev-sidecar.exe

dev-sidecar 工具通过本地代理的方式将https请求代理到一些国内的加速通道上 Gitee仓库下载：https://gitee.com/interesting-goods/dev-sidecaropen in new window

Gitee下载不了的话可以到Github下载：https://github.com/docmirror/dev-sidecar/releasesopen in new window
